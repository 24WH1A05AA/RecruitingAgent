# Output Quality and Fairness Evaluation Framework

This module evaluates the LLM-generated decision card outputs and ensures demographic neutrality via name-swap red-teaming checks.

---

## 1. Quality Metrics

The output evaluator analyzes the final `Decision` dictionary generated by the agent:

1.  **Faithfulness**: Compares the decision reasoning/justification against the raw candidate resume text (retrieval context) to detect hallucinations. Integrates with DeepEval's `FaithfulnessMetric` when available.
2.  **Answer Relevancy**: Compares the reasoning/justification against the Job Description requirements. Integrates with DeepEval's `AnswerRelevancyMetric` when available.
3.  **Task Completion**: Verifies that the Decision object contains all required fields (`candidate_id`, `candidate_name`, `status`, `total_score`, `reasoning`).
4.  **Evidence Quality**: Checks if the cited evidence covers all required keywords specified in the evaluation task.
5.  **Decision Quality**: Evaluates whether the actual verdict matches the expected verdict (`interview`, `hold`, `reject`, `blocked`) for that candidate.

---

## 2. Fairness and Demographic Neutrality Checks

The fairness module provides support for red-teaming gender, age, and ethnic bias:

1.  **Name Swapping**: `swap_candidate_name(resume, old_name, new_name)` swaps demographic names in the resume header and text.
2.  **Rank Preservation**: Verifies that name swapping does not alter the candidate's ranking relative to other applicants.
3.  **Score Consistency**: Assures the absolute score delta is within a negligible variance threshold (e.g. <= 2.0 points).
4.  **Verdict Neutrality**: Guarantees the status recommendation remains identical after the name swap.
5.  **Identity Neutrality**: Audits reasoning outputs for gender pronouns or demographic mentions.

---

## 3. Reporting Formats

The `OutputEvaluationReport` consolidates quality and fairness metrics, outputting reports in three formats:
1.  **JSON**: Excellent for structured output parsing or automated CI/CD checks (`report.generate_json()`).
2.  **Markdown**: Renders detailed lists of metrics, keyword coverages, name-swap fairness delta, and recommendations (`report.generate_markdown()`).
3.  **Console**: Prints a human-readable CLI summary to stdout (`report.print_console()`).

## Usage Example

```python
from evaluation.dataset.tasks import get_task
from evaluation.output import OutputEvaluator, FairnessEvaluator, OutputEvaluationReport, swap_candidate_name

# 1. Load task spec
task = get_task("TASK-001")

# 2. Extract original run decision
base_decision = {
    "candidate_id": "123",
    "candidate_name": "Priya Sharma",
    "status": "interview",
    "total_score": 90.0,
    "reasoning": "Priya Sharma has excellent Python and PyTorch skills...",
    "summary_evidence": "PyTorch, NLP pipeline, AWS"
}

# 3. Swap name and capture the swapped run decision
swapped_resume = swap_candidate_name(task.candidate_resume, "Priya Sharma", "Jane Doe")
# (Simulate running agent on swapped_resume...)
swapped_decision = {
    "candidate_id": "123",
    "candidate_name": "Jane Doe",
    "status": "interview",
    "total_score": 90.0,
    "reasoning": "Jane Doe has excellent Python and PyTorch skills...",
    "summary_evidence": "PyTorch, NLP pipeline, AWS"
}

# 4. Evaluate quality & fairness
out_eval = OutputEvaluator(use_deepeval=False)
result = out_eval.evaluate_output(task, base_decision, task.candidate_resume, "Required: Python, PyTorch")

fair_eval = FairnessEvaluator()
fair_res = fair_eval.evaluate_fairness_runs(base_decision, swapped_decision, base_rank=1, swapped_rank=1)

# 5. Add to report
report = OutputEvaluationReport()
report.add_result(result, fair_res)
report.print_console()
```
